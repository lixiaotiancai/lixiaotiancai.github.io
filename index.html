模块调用了，如果重新定义该函数那么所有引用到该函数的模块都要重改，从而牵一发而动全身。对此我们需要函数的重构，在不影响以前的功能

发表于 2018-04-05

这两天主要就是介绍了一下js模块化。

Day7的练习有一道多选题，还不错，记录一下：
1，模块化规范主要是为了解决命名冲突和模块之间的依赖管理问题。
2，目前通用的模块化规范有CommonJS，AMD，CMD等。
3，模块化规范使开发人员能够以同样的方式来编写模块。
4，模块化规范能够让JS的模块化开发变得简单和自然。

这里总结一下js模块化的发展历程吧：

早在ajax发明之前，js的用途仅仅只是实现一下前端页面简单的逻辑或实现一点简单的动画效果，那时的js代码没有多少，
只需要从上至下执行就可以了。

后来ajax应运而生，前端可以向服务器发出请求并申请返回数据，这就使得js前端的代码量暴涨，很多问题也随之出现，这
其中最严重的问题有：1，全局变量污染、2，命名冲突、3，代码量庞大不利于维护。

这就使得人们不得不思考使用模块化的概念来管理代码，自执行函数算是模块化发展的萌芽。

还有前缀命名法等方法也陆续被人们提出。

直到2009年nodejs的问世，js代码可以运行在服务端，模块开发规范CommonJs也意味着js的模块化开发逐渐开始健壮。以下是
规范的内容：

  1. 模块的标识应遵循的规则（书写规范）
  2. 定义全局函数require，通过传入模块标识来引入其他模块，执行的结果即为别的模块暴漏出来的API
  3. 如果被require函数引入的模块中也包含依赖，那么依次加载这些依赖
  4. 如果引入模块失败，那么require函数应该报一个异常
  5. 模块通过变量exports来向往暴漏API，exports只能是一个对象，暴漏的API须作为此对象的属性。
    当然了这是最初Modules/1.0规范，所以仍存在很多问题，比如外层没有function的包裹，变量全暴露在全局，还有资源的加载
    问题，当然这些问题在后续的版本中都陆续被完善了。

再到后来有了AMD（异步加载所需要的模块）和RequireJS。
AMD规范包含以下内容：
1，用全局函数define来定义模块，用法为：define(id?, dependencies?, factory);
2，id为模块标识，遵从CommonJS Module Identifiers规范
3，dependencies为依赖的模块数组，在factory中需传入形参与之一一对应
4，如果dependencies的值中有”require”、”exports”或”module”，则与commonjs中的实现保持一致
5，如果dependencies省略不写，则默认为[“require”, “exports”, “module”]，factory中也会默认传入require,exports,module
6，如果factory为函数，模块对外暴漏API的方法有三种：return任意类型的数据、exports.xxx=xxx、module.exports=xxx
7，如果factory为对象，则该对象即为模块的返回值

。。。

模块化经历了各种腥风血雨，ECMA终于意识到了问题，终于在ES6中加入了模块标准。

ES6 模块主要具备以下几个基本特点：

1，自动开启严格模式，即使你没有写 use strict
2，每个模块都有自己的上下文，每一个模块内声明的变量都是局部变量，不会污染全局作用域
3，模块中可以导入和导出各种类型的变量，如函数，对象，字符串，数字，布尔值，类等
4，每一个模块只加载一次，每一个 JS 只执行一次， 如果下次再去加载同目录下同文件，直接
从内存中读取。 一个模块就是一个单例，或者说就是一个对象。

关键字有两个，export负责对外开放接口，import负责加载模块。
但现在浏览器对ES6 Module的支持性还有些薄弱。

今天来学习一下有关npm的常用简单命令。

npm install package最简单的的下载包命令，默认下载的是最新版本。

npm install package version下载指定的版本。

npm init在当前目录下生成package.json文件，里面是关于项目的一些配置。

npm install package --s 或 -S把模块的名称及版本信息保存进package.json的dependencies中，也就是
加载项目所依赖的模块信息。当然–s和-S可以省略。这里要注意-S是大写。

npm install package --save-dev 或 -D把模块版本信息加载到package.json的devDependencies,也就是生产
环境依赖中。

npm install package -g 或 --global直接安装模块到全局。

npm uninstall package卸载模块。

当我们没有下载任何模块，但package中的依赖里已配好模块的相关信息，可以直接npm install来安装项目中
所需要的依赖包。
